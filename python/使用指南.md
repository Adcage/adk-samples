# ADK Python 示例使用指南

[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](LICENSE)

<img src="https://github.com/google/adk-docs/blob/main/docs/assets/agent-development-kit.png" alt="Agent Development Kit Logo" width="150">

本文档提供基于 Python [Agent Development Kit](https://github.com/google/adk-python) 构建的示例 Agent 的完整使用指南。这些示例涵盖从简单对话机器人到复杂多 Agent 工作流的各种场景。

## 🚀 快速开始

### 前置要求

1. **安装 Python ADK**

   - 确保已安装并配置 Python Agent Development Kit
   - 参考 [ADK 安装指南](https://google.github.io/adk-docs/get-started/installation/#python) 中的 Python 部分

2. **配置环境变量**

   - 每个 Agent 示例都依赖 `.env` 文件进行配置（如 API 密钥、Google Cloud 项目 ID、区域等）
   - 需要在要运行的 Agent 目录中创建 `.env` 文件（通常通过复制提供的 `.env.example` 文件）
   - 配置这些变量（特别是获取 Google Cloud 凭证）需要仔细操作，详细步骤请参考 [ADK 安装指南的环境设置部分](https://google.github.io/adk-docs/get-started/installation/#python)

3. **Google Cloud 项目（推荐）**
   - 虽然某些 Agent 可能仅使用 API 密钥在本地运行，但大多数会利用 Google Cloud 服务（如 Vertex AI 和 BigQuery）
   - 强烈建议配置 Google Cloud 项目
   - 详细设置请参考 [ADK 快速入门](https://google.github.io/adk-docs/get-started/quickstart/#python)

### 安装步骤

1. **克隆仓库**

   ```bash
   git clone https://github.com/google/adk-samples.git
   cd adk-samples/python
   ```

2. **浏览 Agent 示例**

   - 进入 `agents/` 目录
   - `agents/README.md` 提供了可用 Agent 的概览和分类
   - 浏览子目录，每个目录包含一个特定的示例 Agent 及其 `README.md`

3. **运行 Agent**
   - 从 `agents/` 目录中选择一个 Agent
   - 进入该 Agent 的目录（例如：`cd agents/llm-auditor`）
   - 按照该 Agent 的 `README.md` 文件中的说明进行特定设置（如通过 `poetry install` 安装依赖）并运行 Agent
   - 每个 Agent 和工具都有自己的 `README.md` 文件，包含详细说明

### 注意事项

- 这些 Agent 使用 [Google 模型](https://cloud.google.com/vertex-ai/generative-ai/docs/learn/models) 在 Vertex AI 上构建和测试
- 您也可以使用其他模型测试这些示例
- 请参考 [ADK 教程](https://google.github.io/adk-docs/tutorials/) 了解如何为这些示例使用其他模型

## 📝 代码质量检查

### 本地贡献者预检查

在提交包含 Python 文件更改的 Pull Request 之前，请在本地运行以下脚本，快速验证您的更改是否符合我们的标准。

1. 确保已安装 **Python** 并且脚本可执行（`chmod +x python-checks.sh`）
2. 使用 **run 标志**从仓库根目录运行脚本，指定您修改的 Agent 或 Notebook 文件夹的**相对路径**

| 用途                                     | 命令                                                               |
| :--------------------------------------- | :----------------------------------------------------------------- |
| **运行所有检查**（Black、iSort、Flake8） | `./python-checks.sh --run-all agents/agent_directory_name`         |
| **仅运行 `flake8`**（代码检查）          | `./python-checks.sh --run-lint agents/agent_directory_name`        |
| **仅运行 `black`**（代码格式化）         | `./python-checks.sh --run-black notebooks/notebook_directory_name` |
| **仅运行 `isort`**（导入排序）           | `./python-checks.sh --run-isort notebooks/notebook_directory_name` |
| **获取详细使用说明和选项**               | `./python-checks.sh --help`                                        |

> **注意：** 脚本需要以 `agents/` 或 `notebooks/` 开头的完整相对路径（例如：`agents/academic-research`）。这确保检查严格限定在您正在处理的组件上。

### 自动化检查

我们使用自动化检查来确保所有代码示例的高质量和一致性。脚本将运行 `black`、`isort` 和 `flake8` 来检查格式和代码规范错误。
